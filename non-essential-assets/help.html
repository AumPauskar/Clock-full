Layout

The clock is divided into 3 parts. The first part is the main clock part. This part is only used for displaying time in the form of a 24 hours hour clock. The second part is the stopwatch part which is used for displaying the stopwatch in the form of hours:minutes:seconds. The third part is the timer part, the current plan is to have a json file which decides how long timer will last.

Why so many files???

Now you might ask why so many files in the plan? Well to distribute the workload we have made a set of different filetypes to accomadate different requirements.

1. main.py

Firstly the elephant in the room is main.py, it controls all the major workloads of the program and has all the labels inside it. If you want to see the main core of the program you go to main.py

2. controller.py

Controller.py handles all the side functions of the file for example it handles the information that is passed from the json file to main.py

3. settings.json

settings.json will provide all the information for the main files in main. Mostly settings.json will give the size and text information to the main file and the countdown timer values.

4. README.md

Will provide a small gist of all the modules used and the basic functionality of the application

5. non-essential-assets

This folder will house all the non essential details regarding the application like help, logs and images which are not necessary to run the code properly
